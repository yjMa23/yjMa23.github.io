## 内容原则

- **先讲为什么**：在命令和代码之前，先解释要解决的问题。
- **保持可验证**：示例尽量可以独立运行，并注明前提条件。
- **持续更新**：当实践发生变化时，及时修正文档而不是让它静静过期。
